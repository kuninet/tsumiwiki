# 02. 機能要件

要件IDの凡例: `FR-xx-nn`(FR=Functional Requirement)。優先度は MUST(初期リリース必須)/ SHOULD(初期リリースに入れたい)/ COULD(余裕があれば)。

## 2.1 機能一覧

| 分類 | 機能 | 優先度 |
|---|---|---|
| 認証 | ログイン/ログアウト、ユーザー管理 | MUST |
| 文書管理 | 文書の作成・閲覧・編集・削除・移動・リネーム | MUST |
| エディタ | WYSIWYG Markdown編集、ツールバー、ショートカット | MUST |
| 画像 | ドラッグ&ドロップによる画像挿入 | MUST |
| 整理 | フォルダツリー、タグ付け・タグ絞り込み | MUST |
| 履歴 | 保存ごとの履歴記録、差分表示、過去版復元 | MUST |
| 排他制御 | 編集ロック | MUST |
| 検索 | タイトル・本文の全文検索 | MUST |
| 図表 | Mermaid記法のレンダリング | MUST |
| リンク | 文書間リンク、外部ファイルリンク | SHOULD |
| Obsidian互換 | 既存Obsidianヴォルトの取り込み・記法互換 | MUST |
| ごみ箱 | 削除文書の復元 | MUST |

## 2.2 認証・ユーザー管理

- **FR-AUTH-01 (MUST)** ユーザーはID/パスワードでログインする。未ログインでは文書の閲覧・編集はできない。
- **FR-AUTH-02 (MUST)** 管理者はユーザーの追加・無効化・パスワードリセットができる。
- **FR-AUTH-03 (MUST)** ユーザーには「管理者」「一般」の2ロールを設ける。管理者のみユーザー管理・システム設定が可能。文書の閲覧・編集権限は全ユーザー共通(文書単位の権限制御は行わない)。
- **FR-AUTH-04 (MUST)** パスワードはハッシュ化して保存する(平文保存禁止)。
- **FR-AUTH-05 (SHOULD)** 一定時間操作がない場合はセッションを無効化する(タイムアウト時間は設定可能)。
- **FR-AUTH-06 (MUST)** 初期管理者ユーザーはセットアップ時にCLIで作成する。
- **FR-AUTH-07** パスワードポリシー(最低文字数等)は設けない(空パスワードのみ禁止)。

## 2.3 文書管理

- **FR-DOC-01 (MUST)** ユーザーは任意のフォルダ配下に新規文書を作成できる。
- **FR-DOC-02 (MUST)** 文書はライブラリのルートフォルダ配下にMarkdownファイル(UTF-8、拡張子 `.md`)として保存される。
- **FR-DOC-03 (MUST)** ユーザーは文書のリネーム・フォルダ間移動・削除ができる。
- **FR-DOC-04 (MUST)** フォルダの作成・リネーム・削除ができる。
- **FR-DOC-05 (MUST)** 文書にはタグを複数付与できる。タグは文書ファイル内のフロントマター(YAML)に保存する。
- **FR-DOC-06 (MUST)** 保存時に「誰がいつ編集したか」を記録する(Gitコミットの author として記録)。
- **FR-DOC-07 (MUST)** 文書の削除はごみ箱方式とし、ごみ箱からの復元・完全削除ができる。ごみ箱フォルダはObsidianと互換の `.trash/` を使用する。
- **FR-DOC-08 (MUST)** サーバー上のライブラリフォルダに外部ツールで直接置かれた/編集されたMarkdownファイルも、Wiki上で認識・閲覧できる。生成AIエージェント(Cowork等)がライブラリフォルダの文書を直接作成・修正する運用を正式にサポートする。
  - 変更はファイルシステム監視により自動検知し(数秒〜数十秒)、履歴(syncコミット)・検索インデックス・表示中のツリーへ自動反映する。
  - 監視が効かない環境(ネットワークドライブ等)の保険として、画面に手動の「更新確認」操作も設ける。
  - Wikiで編集ロック中の文書が外部から変更された場合は、Wiki側の保存時に競合を検知し、ユーザーへ再読み込みを促す(上書き喪失を防ぐ)。

## 2.4 エディタ

- **FR-EDIT-01 (MUST)** 編集中、Markdownは即座にWYSIWYGレンダリングされる(Obsidianのライブプレビュー相当)。
- **FR-EDIT-02 (MUST)** Markdown記法の直接入力に対応する。対応記法(最低限): 見出し(#)、太字/斜体/打消し、箇条書き/番号付きリスト、チェックリスト、表、コードブロック(シンタックスハイライト)、引用、水平線、リンク、画像。
- **FR-EDIT-03 (MUST)** ツールバーからワープロ感覚で書式指定できる(見出しレベル、太字、斜体、リスト、表挿入、リンク挿入等)。
- **FR-EDIT-04 (MUST)** キーボードショートカットに対応する。最低限: `Ctrl/Cmd+S`(保存)、`Ctrl/Cmd+B`(太字)、`Ctrl/Cmd+I`(斜体)、`Ctrl/Cmd+Z / Shift+Z`(取り消し/やり直し)、`Ctrl/Cmd+K`(リンク挿入)。
- **FR-EDIT-05 (MUST)** 日本語入力(IME)で正常に編集できる。IME変換中の文字列が確定前に書式処理されない等、IME事象に配慮する。
- **FR-EDIT-06 (MUST)** 編集内容はMarkdownとして破壊的変換なく保存される(WYSIWYG⇔Markdownの往復で意図しない差分が発生しないこと)。
- **FR-EDIT-07 (SHOULD)** 未保存の変更がある状態で画面を離れようとした場合、警告を表示する。
- **FR-EDIT-08 (SHOULD)** 一定間隔での自動保存(下書き)を行う。自動保存はGit履歴を汚さない方式とする(明示保存時のみコミット)。
- **FR-EDIT-09 (COULD)** Markdownソース表示モードへの切り替えができる。
- **FR-EDIT-10 (MUST)** Mermaidコードブロック(` ```mermaid `)を図としてレンダリングする(閲覧モードおよび編集中のプレビュー)。Obsidianの標準Mermaid対応と互換とする。

## 2.5 画像・添付

- **FR-IMG-01 (MUST)** 画像ファイルをエディタへドラッグ&ドロップすると、編集中の文書と同じフォルダに保存され、文書に画像として挿入される(既存Obsidianヴォルトの運用に合わせる。保存先は設定で変更可能)。
- **FR-IMG-02 (MUST)** クリップボードからの画像貼り付け(スクリーンショット等)にも対応する。
- **FR-IMG-03 (MUST)** 画像はMarkdown標準の画像記法(相対パス)で参照し、外部ツールでも表示互換性を保つ。
- **FR-IMG-04 (COULD)** 画像以外のファイル(PDF等)の添付にも対応する。

## 2.6 整理・ナビゲーション

- **FR-NAV-01 (MUST)** フォルダペインにライブラリのフォルダ階層をツリー表示し、文書を選択して開ける。
- **FR-NAV-02 (MUST)** タグペインに全タグの一覧(件数付き)を表示し、タグ選択で該当文書を絞り込める。
- **FR-NAV-03 (MUST)** タイトルおよび本文の全文検索ができる。検索結果はヒット箇所のプレビュー付きで表示する。
- **FR-NAV-04 (SHOULD)** 最近更新された文書の一覧を表示する。

## 2.7 リンク

- **FR-LINK-01 (SHOULD)** 文書から他の文書へ内部リンクを張れる(エディタ上で文書名を補完入力できる)。リンク記法はObsidian互換の `[[文書名]]`(wikilink)を第一とする(→ 2.10)。
- **FR-LINK-02 (SHOULD)** PC/Macファイルシステムや外部ドライブ上のファイルへのリンク(`file://` 形式等)を記載できる。
  - ブラウザのセキュリティ制約により `file://` リンクはクリックで直接開けないため、リンク上に「パスをコピー」ボタンを提供する。加えて、ブラウザのポリシー設定/拡張で `file://` 遷移を許可した環境では直接開けることを妨げない。
- **FR-LINK-03 (COULD)** 内部リンクのリンク切れ(リンク先文書の削除・リネーム)を検出・表示する。

## 2.8 履歴管理

- **FR-HIST-01 (MUST)** 文書の保存のたびに履歴を記録する(Gitコミット)。コミットauthorにはログインユーザーを設定する。
- **FR-HIST-02 (MUST)** 文書ごとに履歴一覧(日時・編集者)を表示できる。
- **FR-HIST-03 (MUST)** 過去の版の内容を閲覧し、現在の版との差分を表示できる。
- **FR-HIST-04 (MUST)** 過去の版の内容を現在の文書として復元できる(履歴は巻き戻さず、復元操作も新しい履歴として記録する)。

## 2.9 Obsidian互換(既存ヴォルトの取り込み)

既存のObsidianヴォルト(データフォルダ)をライブラリルートにコピーするだけで、そのままWikiとして利用できることを目標とする。

- **FR-OBS-01 (MUST)** Obsidianヴォルトをライブラリルートにコピーした状態で、フォルダ階層・文書・フロントマターのタグが認識され、閲覧・編集できる。
- **FR-OBS-02 (MUST)** Obsidianのwikilink記法 `[[文書名]]`・`[[文書名|表示名]]` をリンクとしてレンダリングし、クリックで該当文書へ遷移できる。
- **FR-OBS-03 (MUST)** Obsidianの埋め込み記法 `![[画像ファイル名]]` を画像として表示できる。
- **FR-OBS-04 (MUST)** `.obsidian/` フォルダ、`.trash/` 等の隠しフォルダは文書ツリーに表示せず、内容を破壊しない(Gitコミット対象からも除外)。
- **FR-OBS-05 (MUST)** 添付ファイルの保存先は既定で「文書と同じフォルダ」とし(既存ヴォルトの設定に合致)、設定で変更可能とする。
- **FR-OBS-06 (MUST)** フロントマターの `tags:` と本文中のインラインタグ(`#タグ`)の両方をタグとして認識し、タグペインの集計対象に含める(既存ヴォルトで両方使用されているため)。
- **FR-OBS-07 (SHOULD)** 本システムで編集・保存した文書は、Obsidianで開いても互換性が保たれる(記法・フロントマターを破壊しない)。

- **FR-OBS-08 (MUST)** Dataviewブロック(` ```dataview `・` ```dataviewjs `)はクエリを実行せず、コードブロックとしてそのまま表示する。本システムでの編集・保存によってブロック内容を破壊しない(Obsidianに戻したとき従来どおり動作すること)。

注: Dataviewのクエリ実行は初期リリースでは対象外(→ [07_将来要件](07_将来要件.md) FUT-07)。コールアウト(`> [!note]`)等その他のプラグイン記法は既存ヴォルトで未使用のため対応しない(Mermaidのみ FR-EDIT-10 で対応)。

## 2.10 排他制御(編集ロック)

- **FR-LOCK-01 (MUST)** ユーザーが文書の編集を開始すると、その文書はロックされる。他ユーザーは閲覧のみ可能で、「○○さんが編集中」であることが表示される。
- **FR-LOCK-02 (MUST)** 保存して編集を終了する、または編集を破棄するとロックは解除される。
- **FR-LOCK-03 (MUST)** ブラウザクラッシュ等でロックが残留した場合に備え、無操作一定時間(例: 30分)でロックを自動解放する。
- **FR-LOCK-04 (SHOULD)** 管理者は残留ロックを手動で強制解除できる。
