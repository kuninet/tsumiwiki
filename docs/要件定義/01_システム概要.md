# 01. システム概要

## 1.1 目的

ObsidianのようなWYSIWYG Markdownエディタを備えた、複数人で利用できるWebベースのWikiシステム「TsumiWiki」を構築する。文書はサーバー上の指定フォルダ配下に素のMarkdownファイル(UTF-8)として保存し、特定アプリケーションへのロックインを避ける。

## 1.2 背景・課題認識

- チーム内のナレッジをMarkdownで蓄積・共有したい。
- Obsidianはローカルアプリであり、複数人での共同利用・編集者の記録に難がある。
- データはツール独自形式ではなく、ファイルシステム上のMarkdownとして残したい(将来の移行容易性)。

## 1.3 システム化の範囲(スコープ)

### 対象(初期リリース)

- Webブラウザ(PC/Mac)で動作するWikiアプリケーション
- WYSIWYG Markdownエディタ(Markdown記法入力 + ツールバー書式指定)
- フォルダツリー・タグによる文書の整理と閲覧
- タイトル・本文の全文検索
- ユーザーログインと編集者の記録
- 画像のドラッグ&ドロップ挿入
- Gitベースの文書履歴管理と過去版への復元
- ごみ箱(削除文書の復元)
- Mermaid記法による図のレンダリング
- 既存Obsidianヴォルトの取り込み(コピーするだけで利用可能)
- 単一の文書ライブラリ(システムで1つ)

### 対象外(将来要件 → [07_将来要件](07_将来要件.md))

- AWS等クラウド環境への本格デプロイ
- 「個人」「共用」等の複数ライブラリ
- Draw.io的な作図機能(Mermaid記法レンダリングは初期リリースに含む)
- リアルタイム共同編集(Googleドキュメント方式)
- モバイル(スマートフォン/タブレット)対応

## 1.4 利用者・利用環境

| 項目 | 内容 |
|---|---|
| 想定ユーザー数 | 〜20人程度の小規模チーム |
| 同時利用 | 数人程度の同時アクセスを想定 |
| クライアント環境 | PC/MacのモダンブラウザChrome / Edge / Safari / Firefox 最新版 |
| サーバー環境 | 社内のWindows PC(将来AWSを視野) |
| 言語 | 日本語(UI・文書とも日本語を第一級で扱う) |

## 1.5 用語定義

| 用語 | 定義 |
|---|---|
| 文書(ページ) | Wiki上の1つのMarkdownファイルに対応する編集単位 |
| ライブラリ | 文書一式を格納するルートフォルダ。Gitリポジトリとして管理される |
| WYSIWYG | 編集中にMarkdownがリアルタイムにレンダリングされ、見た目のまま編集できる方式 |
| 編集ロック | ある文書を1ユーザーが編集中の間、他ユーザーの編集を禁止する排他制御 |
| タグ | 文書に付与する分類ラベル。1文書に複数付与可能 |
