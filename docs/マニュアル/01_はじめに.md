# 01. はじめに

## TsumiWikiとは

TsumiWiki(積みウィキ)は、ObsidianライクなWYSIWYG Markdownエディタを備えた **チーム向けWebベースWikiシステム** です。文書はサーバー上のフォルダに素のMarkdownファイル(UTF-8)として保存され、既存のObsidianヴォルトをコピーするだけでそのまま使えることを目指しています。

> 名前の由来は「知識を積む」+ Wiki。発音は「積み木」に掛けています。

## 特徴

- 🖊️ **WYSIWYG Markdownエディタ** — 書きながらそのまま整形された姿を確認できます。Markdown記法もツールバーも両方使えます。
- 👥 **複数人で使える** — ログイン・編集者の記録・編集ロックで安心して同時利用できます。
- 📁 **素のMarkdownで保存** — 文書の実体はフォルダ上の `.md` ファイル。特定ツールに縛られません。
- 🔮 **Obsidian互換** — 既存ヴォルトをコピーするだけで動作。`[[wikilink]]`・`![[埋め込み]]`・`#タグ` に対応します。
- 🕘 **Git履歴** — 保存のたびにGitコミットされ、差分表示・過去版への復元ができます。
- 🏷️ **フォルダとタグ** — フォルダツリーとタグの両方で文書を整理できます。
- 🔍 **日本語全文検索** — trigramベースの検索で日本語も直接検索できます。
- 🖼️ **画像ドラッグ&ドロップ** — 画像は本文にドロップまたは貼り付けするだけで挿入されます。
- 📊 **Mermaid図** — コードブロックで書けばそのまま図としてレンダリングされます。
- 📝 **テンプレート・デイリーノート** — よく使う雛形を登録して新規作成に使えます。日々の日誌もワンクリック。
- 🌗 **ライト/ダークテーマ** — お好みで切替できます。

## 想定利用シーン

- 小規模チーム(数名〜20名程度)の社内Wiki
- 議事録・仕様書・手順書などのナレッジベース
- 既存のObsidianヴォルトのWeb共有
- 生成AIエージェント(Claude Code / Cowork等)による文書の直接更新の受け皿(TsumiWikiが外部変更を自動取り込みします)

## 用語

このマニュアルで使う用語をまとめます。

| 用語 | 意味 |
|---|---|
| **ライブラリ** | 文書を保存するフォルダの総称。1つのTsumiWikiに1ライブラリ。 |
| **文書** | 拡張子 `.md` のMarkdownファイル1つ。ページに相当。 |
| **フォルダ** | 文書を整理する入れ物。階層構造を持てます。 |
| **フロントマター** | 文書の先頭にある `---` で囲まれたYAML部分。タグや作成日を保存します。 |
| **タグ** | 文書に付ける分類ラベル(例: `#議事録`)。frontmatter または本文中の `#タグ` として保存されます。 |
| **編集ロック** | 「今この文書を編集中」の目印。他のユーザーはロック中の文書を閲覧のみできます。 |
| **下書き** | 保存前の一時的な内容。30秒ごとに自動でサーバーに退避され、次回の編集開始時に復元できます。 |
| **テンプレート** | 新規文書の雛形。`{{date}}` 等の変数を含めることができます。 |
| **デイリーノート** | 「今日の日誌」文書。日付ごとに1文書を自動で用意します。 |
| **ゴミ箱** | 削除された文書の一時置き場(`.trash/` フォルダ)。復元も可能です。 |
| **ウィキリンク** | `[[文書名]]` 形式のリンク。Obsidian互換です。 |
| **埋め込み** | `![[画像.png]]` 形式で画像等を本文に埋め込む記法。 |
| **管理者(admin)** | ユーザー管理・ライブラリ設定・ゴミ箱完全削除ができる特別なロール。 |

## 動作環境

- **サーバー**: Windows PC(または Linux/macOS)、Node.js 20.19以上、Git for Windows
- **クライアント**: モダンなWebブラウザ(Chrome/Edge/Firefox/Safari)。最小画面幅 1280px 推奨
- **推奨規模**: 〜20名程度のチーム
