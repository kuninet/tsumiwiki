# 03. エディタとMarkdown

TsumiWikiのエディタは Obsidian のライブプレビュー編集に近い感覚で、Markdownを書きながら整形済みの姿を確認できます。閲覧モードと編集モードで見た目は完全に一致します。

## 3.1 ツールバー

編集モードでのみ表示されます。カーソル位置の状態がボタンのアクティブ表示に反映されます。

| ボタン | 動作 |
|---|---|
| **H1 / H2 / H3** | 見出しレベルのトグル |
| **B** | 太字(Ctrl/Cmd+B) |
| **I** | 斜体(Ctrl/Cmd+I) |
| **S** | 打消し |
| **• 箇条書き** | ビュレットリスト |
| **1. 番号リスト** | 番号付きリスト |
| **☑ チェック** | タスクリスト(`- [ ]` / `- [x]`) |
| **表** | 3×3 のGFM表を挿入(ヘッダ行あり) |
| **コード** | フェンス付きコードブロック |
| **引用** | ブロック引用(`>`) |
| **区切り線** | 水平線(`---`) |
| **リンク** | リンク挿入ダイアログ(Ctrl/Cmd+K) |
| **画像** | ファイル選択でアップロード → `![[ファイル名]]` を挿入 |
| **Mermaid** | 空のMermaidコードブロックを挿入 |
| **テンプレ適用** | テンプレート選択ダイアログを開き、挿入または追記([04章 4.1](04_便利機能.md#41-テンプレートから新規文書を作成する)) |

## 3.2 主なMarkdown記法

以下は Markdown 標準 / GFM 記法です。すべてツールバーからも操作できます。

```markdown
# 見出し1
## 見出し2

**太字** *斜体* ~~打消し~~

- 箇条書き
- 二つ目
  - ネスト

1. 番号付き
2. 番号付き

- [ ] タスク未完
- [x] タスク完了

> 引用ブロック

`インラインコード` と

​```typescript
// コードブロック(言語指定)
​```

| 列1 | 列2 |
|---|---|
| a | b |

[通常リンク](https://example.com)
![alt付き画像](画像のパス.png)
```

コードブロック内の `#` はタグとして扱われません。

## 3.3 Obsidian互換の記法

TsumiWikiは以下のObsidian互換記法を保全します。往復変換テストで「開いて保存しても記法が壊れない」ことを保証しています。

### ウィキリンク `[[...]]`

- `[[文書名]]` — 該当文書へのリンク
- `[[フォルダ/文書名]]` — パス指定
- `[[文書名|別名]]` — 表示テキストを別名にする

**入力補完**: `[[` を入力すると候補ポップアップが開き、全文書からタイトル・パスの部分一致で最大20件表示されます。**↑↓ / Enter / Escape** で操作、日本語入力の変換中は発火しません。

閲覧モードでリンクをクリックすると該当文書に遷移します。リンク先が存在しない場合は「リンク先が見つかりません」トーストが表示されます。

### 埋め込み `![[...]]`

`![[filename.png]]` の形で画像や添付ファイルを本文中に埋め込めます。

- 対応拡張子: `png` / `jpg` / `jpeg` / `gif` / `svg` / `webp`
- 解決順: **同フォルダ → ライブラリルート → attachments/** の順に探索します
- 画像以外(文書埋め込み等)は現状リンク表示のみ(展開されません)

### インラインタグ `#タグ`

本文中の `#タグ名` は自動でハイライトされ、タグとして集計されます。

- `#` の直後に空白・記号を含まない文字列
- 行頭または空白の直後、`(` の直後のみ
- コードブロック・インラインコード内は除外
- 数字のみのタグは除外(例: `#123` は日付やIDと誤検出を避けるためタグにしません)

保存時に自動でフロントマターの `tags` フィールドとも統合されます。文書上部の **タグチップエディタ** でも同じタグを編集できます([04章 4.3](04_便利機能.md#43-タグを編集する))。

## 3.4 画像の挿入

以下のいずれの方法でも、TsumiWikiが自動でサーバーにアップロードし `![[filename]]` として本文に挿入します。

1. **ドラッグ&ドロップ**: 画像ファイルをエディタにドロップ(複数まとめて可)
2. **クリップボード貼り付け**: スクリーンショット等をコピーして Ctrl/Cmd+V
3. **ツールバー画像ボタン**: ファイル選択ダイアログから

- 対応形式: `png` / `jpg` / `jpeg` / `gif` / `svg` / `webp`
- サイズ上限: 20MB(サーバー設定で変更可)
- 保存先: **文書と同じフォルダ**(既定。管理者設定で変更可)
- ファイル名: `image-YYYYMMDDHHmmss.png` 形式(自動命名)

アップロードは1つの操作としてGitにコミットされます(`attach:` プレフィックス)。

## 3.5 Mermaid図

コードブロックの言語を `mermaid` にすると、閲覧時にSVGとしてレンダリングされます。ツールバー **[Mermaid]** ボタンで空ブロックを挿入できます。

​```mermaid
graph LR
  A[開始] --> B{条件}
  B -->|Yes| C[処理]
  B -->|No| D[終了]
​```

- カーソルがブロック外にあるときはプレビュー、内側にフォーカスするとソース編集
- 構文エラー時はエラーメッセージ+ソース表示
- プレビューをクリックするとソースへフォーカスが移動します
- セキュリティ: `securityLevel: strict` で実行(スクリプトタグ等は無効)

## 3.6 リンクの種類

TsumiWikiでは以下の3種類のリンクを扱えます。

| 記法 | 例 | 閲覧時のクリック動作 |
|---|---|---|
| ウィキリンク | `[[議事録/2026-07-08]]` | 内部の該当文書へ遷移 |
| 外部URL | `[GitHub](https://github.com/)` | 新規タブで開く(`_blank noopener`) |
| ファイルパス | `[設計書](file:///C:/docs/design.pdf)` または `[共有](\\\\fileserver\\share)` | クリック時にパスをクリップボードにコピー(直接遷移は環境依存) |

許可されるURLスキームは `http` / `https` / `mailto` / `file` のみです。

## 3.7 空リストの解除

Obsidianと同様、空のリスト項目末尾で **Backspace** を押すとリストが解除され、通常の段落に戻ります。

## 3.8 日本語入力(IME)

- 日本語変換中はショートカット・入力規則・候補ポップアップの発火が抑制されます(変換中に誤発火しません)
- macOS(ライブ変換)・Windows(MS-IME)で確認済み

## 3.9 未知の記法の保全(重要)

パースできないブロック(HTMLブロック等)は **等幅・グレー背景の raw ブロック** として原文のまま保持されます。TsumiWikiは「読んだものは壊さない」ことを最優先とし、往復変換テストで冪等性を担保しています。

Obsidian特有の記法(Dataviewブロックなど)は現状 **実行せず** 通常のコードブロックとして表示されますが、原文は失われません。

## 3.10 保存時のフロントマター

- エディタで扱うのは本文のみで、フロントマター(`---` 〜 `---`)はサーバー側で分離管理されます
- タグ編集は文書ヘッダーの **タグチップエディタ** で行います(本文中の `#タグ` とも連動)
- 未知のフロントマターキー(例: `created`, `updated`, カスタムキー)は保存時に順序・コメント・スタイルを保持したまま残ります
