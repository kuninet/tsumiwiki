# 04. 便利機能

基本操作とエディタを一通り把握したら、TsumiWikiの効率化機能を活用しましょう。

## 4.1 テンプレートから新規文書を作成する

会議録・日報・仕様書など、よく使う文書の雛形を **テンプレート** として登録しておくと、ワンクリックで新規作成できます。

### 起動

サイドバーフッターの **[📄 テンプレから新規]** をクリックします。

### 2段のダイアログ

**Step 1: テンプレート選択**

- テンプレート一覧が表示されます
- 入力欄でインクリメンタル絞り込み(名前・パスの部分一致)
- ↑↓ / Enter / Escape で操作
- 選択すると Step 2 に進みます

**Step 2: タイトルと作成先**

- **タイトル** — 新規文書のファイル名(拡張子不要)
- **作成先フォルダ** — 空欄でライブラリ直下。テンプレ側で `target_folder` が指定されていれば既定値になります
- **[作成]** で新規文書が作成され、そのまま開きます

### テンプレート内の変数

テンプレート本文中に以下の変数を書いておくと、作成時に自動で置換されます。

| 変数 | 展開結果 |
|---|---|
| `{{date}}` | 今日の日付(既定 `YYYY-MM-DD`)。書式指定は `{{date:YYYY年MM月DD日}}` のように |
| `{{time}}` | 現在時刻(既定 `HH:mm`) |
| `{{year}}` `{{month}}` `{{day}}` `{{hour}}` `{{minute}}` | 個別のコンポーネント |
| `{{title}}` | 新規文書のタイトル |
| `{{user}}` | 作成者の表示名 |
| `{{cursor}}` | 作成後にカーソルを置く位置マーカー(例: `## 議題\n\n{{cursor}}`) |

日付書式トークン: `YYYY`(西暦4桁)/ `YY`(2桁)/ `MM`(月)/ `DD`(日)/ `HH`(24時制)/ `mm`(分)/ `ss`(秒)/ `aaaa`(月曜日)/ `aaa`(月)/ `dddd`(Monday)/ `ddd`(Mon)

未知の変数(タイポ含む)はそのまま残ります。

### テンプレートの登録場所

管理者がライブラリ設定で指定した **テンプレフォルダ** の直下(既定 `_templates`)に `.md` ファイルを置くだけで自動で認識されます。テンプレート自身のフロントマターに以下を書けます:

```yaml
---
target_folder: 議事録         # 既定の作成先フォルダ(任意)
description: 定例会議の議事録  # ダイアログの補助説明(任意)
---
```

これらのメタキーは新規文書には残らず、テンプレ側のみに書けます。

## 4.2 今日の日誌を作る

日付ごとに1つのデイリーノートを作る運用に便利です。

### 起動

サイドバーフッターの **[📓 今日の日誌]** をクリックします。

- 今日のノートがまだ無い場合: 自動で作成して開きます(「今日の日誌を作成しました」トースト)
- 既に存在する場合: そのまま開きます(「今日の日誌を開きました」情報トースト)

### 設定

作成先フォルダ・適用テンプレート・ファイル名パターンは **管理者のライブラリ設定** で決まります([05章 5.3.2](05_管理者ガイド.md#532-デイリーノート))。

- ファイル名パターン例: `YYYY-MM-DD` → `2026-07-08.md`
- 曜日付きの例: `YYYY-MM-DD(aaa)` → `2026-07-08(水).md`

### 「今日」の判定

サーバーのローカルタイムゾーンを使います(全ユーザーが同一TZ運用の前提)。

## 4.3 タグを編集する

文書のタイトル下に表示されている **タグチップ** から編集できます。

### 閲覧モード
- タグをクリック → タグペインに切替 & そのタグで絞り込み

### 編集モード
- **`#タグ名`** をクリック → インライン入力に変わってリネーム
- チップ右の **×** → タグ削除
- 末尾の **[+ タグを追加]** → 新規追加

タグ名のルール:
- 先頭の `#` は自動で落とされます
- 使える文字: 文字・数字・`_` / `/` / `-` のみ(NFC正規化)
- 最大30文字

チップから追加/変更/削除したタグは、**本文中の `#タグ` 表記および frontmatter の `tags` フィールドの両方に反映されます**。

## 4.4 テンプレート適用(既存文書に流し込む)

すでに開いている文書に、テンプレートを追加で挿入できます。

1. 編集モードでエディタツールバーの **[テンプレ適用]** ボタンをクリック
2. Step 1 でテンプレートを選択
3. Step 2 で **[挿入]** または **[追記]** を選択
   - **挿入** — 現在のカーソル位置に流し込みます
   - **追記** — 文書末尾に追加します
4. テンプレート内の `{{cursor}}` があれば、挿入後にその位置へカーソルが移動します

1つのUndoで完全に取り消せる単位で挿入されます。

## 4.5 複数の文書・フォルダを一括で扱う

大量の整理をするときの効率化テクニックです。

### 複数選択

- **Ctrl/Cmd+クリック** — 個別に追加/解除
- **Shift+クリック** — 範囲選択

選択が2件以上あるとサイドバー上部に「N件選択中」バーが出ます。

### 一括の操作

- **ドラッグ&ドロップ** — 選択中のもの全てが同時に移動
- **右クリック → 「選択したものを新規フォルダに移動(N件)」** — 新しいフォルダを作って一発でまとめる
- **上部バー → [+ 選択したものを新規フォルダに移動]** — 同じ操作

失敗した項目がある場合は「◯件を移動しました。◯件が失敗しました」の警告トーストが出ます。自分自身や子孫への移動、同じ親への移動はブロックされます。

## 4.6 ウィキリンクの候補ポップアップ

編集中に `[[` を入力すると、全文書からタイトル・パスの部分一致で候補が最大20件表示されます。

- **↑ / ↓** で候補移動
- **Enter** で決定
- **Escape** で閉じる
- 日本語変換中は発火しない

## 4.7 リンクをキーボードで挿入する

編集モードで **Ctrl/Cmd+K** を押すとリンクダイアログが開きます。

- 選択中のテキストがある場合: その範囲をリンク化
- 選択なしの場合: URL自体がリンクテキストとして挿入されます

## 4.8 検索を素早く使う

- **Ctrl/Cmd+K**(閲覧時)でヘッダー検索ボックスにフォーカス
- 未入力時は「最近開いた文書」が候補に出るため、直近読んだ文書へすぐ戻れます
- 検索結果の右側にタグ候補も出るため、タグ絞り込みへの導線としても使えます

## 4.9 テーマ切替

ヘッダー右の ☾ / ☀ アイコンで **ライト/ダークテーマ** を切り替えられます。選択はブラウザに保存され、次回開いたときも維持されます。

## 4.10 外部変更の取り込み(更新確認)

TsumiWikiは、外部ツール(Obsidian・生成AIエージェント・直接ファイル操作)がライブラリを変更したときも、以下の経路で自動で取り込みます。

- ファイル監視(chokidar)によるリアルタイム反映
- 定期ポーリング(5分間隔)
- 文書取得時のmtime不一致検知

自動反映を待たずに今すぐ取り込みたいときは、ヘッダー右の **[↻ 更新確認]** ボタンを押します。ツリーとタグが再取得され「更新を確認しました」トーストが表示されます。

## 4.11 個人設定(パスワード変更)

右上のユーザーメニュー → **[設定]** で `/settings` を開きます。

- アカウント情報(ユーザーID / 表示名 / ロール)を確認できます
- パスワード変更が可能です
- 変更に成功すると、**現行以外の全セッションが失効します**(不要な端末の一括ログアウトに使えます)
