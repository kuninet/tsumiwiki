# 07. トラブルシューティング

よくあるトラブルとその対処法をまとめます。

## 7.1 ログイン関連

### ログインできない(パスワードは合っているはず)

- **レート制限**: 15分の間に10回失敗すると、そのIP+ユーザーID組み合わせがブロックされます。しばらく待ってから再試行してください
- **ユーザーが無効化されている**: 管理者に確認してもらってください
- **パスワードを忘れた**: 管理者に依頼して `/admin/users` からリセット([05章 5.1.4](05_管理者ガイド.md#514-パスワードリセット))

### 突然ログアウトされる

以下のいずれかで、あなたのセッションが失効した可能性があります。

- 管理者があなたのパスワードをリセット、または無効化・降格した
- あなた自身が別ブラウザ・別端末で **パスワードを変更した**(現行以外は全失効の仕様)
- セッションTTLタイムアウト(既定8時間)

再ログインすれば問題ありません。

## 7.2 保存関連

### 保存できない、「保存の競合」ダイアログが出る

他のユーザーが先にその文書を保存したときに発生します。

- **[自分の内容で上書き保存]** — 自分の変更を優先(相手の変更は履歴には残るので後で差分から見られます)
- **[破棄して最新を読み込む]** — 自分の変更を諦めてサーバー最新に切り替える

慎重に決めたい場合は、上書き前に別タブで最新版と履歴を確認してから決めましょう。

### 「ロック期限切れ」で閲覧モードに戻された

長時間何も操作していないと、サーバー側で編集ロックが自動解放されます(既定30分)。

- 大丈夫です。もう一度編集を開始すれば新しいロックが取れます
- ただし、未保存の変更は下書きとして保存されているため、再開時に「未保存の下書きがあります。復元しますか?」ダイアログで復元できます

### 「◯◯さんが編集中」で編集できない

他のユーザーが編集ロックを取得しています。

- しばらく待って、相手が保存または終了するとロックが解放されます
- 相手が席を離れてPCも操作されていない場合、30分後に自動解放されます
- 緊急時は管理者に **強制ロック解除** を依頼してください([05章 5.4](05_管理者ガイド.md#54-編集ロックの強制解除))

### 下書きが復元されない

- 別のブラウザ・別のPCで編集していた場合、下書きはそのブラウザ/PCのユーザーに紐づいて保存されています(サーバー上には残るため、同じユーザーで同じ文書を開けば復元プロンプトが出ます)
- 下書きは編集ロックを取得できる場合のみ書き込まれます。他ユーザーがロック中の間は下書きは更新されません

## 7.3 検索関連

### 検索結果が出ない

- **3文字未満のクエリ**: TsumiWikiはtrigram(3文字組)で検索するため、日本語は3文字以上入れると結果が安定します。検索ボックス下に「3文字以上を推奨します」と表示されます
- **タグは別枠**: タグは検索結果とは別のセクション(右側)に表示されます。タグを探しているならタグペインの方が確実です
- **未反映**: サーバー起動直後や大量ファイルのインポート直後はインデックスがまだ完了していない可能性があります。数分待つか、`tsumiwiki reindex` を実行

### 特定の文書がツリーに出ない

- ファイル名が `.md` 拡張子か確認してください
- ドットで始まるフォルダ(`.hidden` など)は除外されます
- `.obsidian/` `.tsumiwiki/` 配下のファイルは常に非表示です
- ヘッダー **[↻ 更新確認]** ボタンで再スキャンしてみてください

## 7.4 画像・添付関連

### 画像がアップロードできない

- **対応拡張子**: `png` / `jpg` / `jpeg` / `gif` / `svg` / `webp` のみ
- **サイズ上限**: 既定20MB。それ以上は413エラーで拒否されます
- **保護パス**: `.trash/` `.obsidian/` `.tsumiwiki/` `.git/` 直下の文書に対する添付はブロックされます

### 埋め込み画像が表示されない(`![[filename]]` が壊れる)

- ファイルの解決順は **同フォルダ → ライブラリルート → attachments/** です
- どこにも見つからないと画像がロードできません
- ファイル名の大文字・小文字が一致しているか確認してください

### アップロードした画像がGitに載らない

- 添付はアップロード時に自動でGitコミット(`attach:` プレフィックス)されます
- コミットが載っていない場合、サーバー側のGit設定に問題がある可能性があります
- サーバーログ(`LOG_FILE` の内容)を確認してください

## 7.5 ウィキリンク・埋め込み

### `[[リンク]]` が「リンク先が見つかりません」トーストを出す

- 該当タイトルの文書が存在しない、または同名文書が複数あって最短パスで解決されていない可能性があります
- 曖昧な場合は `[[フォルダ/文書名]]` のようにパス付きで書くと確実です

### `[[` を入力しても候補が出ない

- 日本語入力の変換中は候補ポップアップが発火しません(仕様)。確定してからもう一度 `[[` を入力してみてください

## 7.6 テンプレート・デイリーノート

### 「テンプレから新規」を押してもテンプレが選べない

- 管理者がライブラリ設定でテンプレフォルダを設定していない、またはフォルダに `.md` ファイルが無い可能性があります
- テンプレフォルダは既定 `_templates`。管理者に確認してください([05章 5.2.1](05_管理者ガイド.md#521-テンプレート設定))

### 「今日の日誌」を押してもエラーが出る

- 管理者がデイリーノート設定を完了していない可能性があります
- 特に **ファイル名パターン** に `{{...}}` 変数構文を書くとバリデーションエラーになります。素の日付書式(例: `YYYY-MM-DD`)にしてもらってください

### テンプレ内の `{{変数}}` が展開されない

- 未知の変数名(タイポ含む)はそのまま残る仕様です。使える変数名は [04章 4.1](04_便利機能.md#41-テンプレートから新規文書を作成する) の一覧で確認してください
- 日付書式の中で使う日は **大文字の `DD`** です。`dd`(曜日短)や `mm`(分)と間違えていないか確認してください

## 7.7 履歴・差分

### 「この版に戻す」を押しても何も起きない

- 編集中の場合は「未保存の変更があります。破棄してもよろしいですか?」ダイアログが表示されます
- ロックを取得できない場合(他ユーザーが編集中)、復元操作は失敗します

### 過去版を復元したら履歴が消えるのでは?

いいえ。復元は「その内容を新しくコミット」する形で行われるため、履歴自体は改変されません(`restore:` プレフィックスの新規コミットが増えます)。

## 7.8 タグ関連

### タグが二重に登録されている

- 本文の `#タグ` と frontmatter の `tags` フィールドの両方に同じ名前があると片方だけに表示されないタイミングが一瞬あります。保存後は自動でマージされます
- タグ名は NFC 正規化されるため、見た目が同じでも正規化違いの2種類が別タグ扱いになる場合があります(通常のIMEでは発生しません)

### タグが認識されない

- 対応文字: 文字・数字・`_` / `/` / `-` のみ
- `#123` のような数字のみは日付やIDと誤検出防止のため除外されます
- コードブロック内の `#` はタグとして扱われません

## 7.9 サーバー運用関連(管理者向け)

### バックアップpushが失敗する

- サーバーログ(`LOG_FILE`)を確認
- Windowsサービスとして動かしている場合、NSSMの実行ユーザーがUNCパスにアクセスできるか(`cmdkey`)を確認
- `git config --global core.autocrlf false` になっているか(Windows)

### 保存がやたら遅い

- Gitの直列キューが混雑している可能性(小規模チームでは通常問題になりません)
- ライブラリがネットワークドライブ上にある場合、Gitオペレーションが遅くなります。ローカルドライブへの配置を推奨

### 外部変更が反映されない

- ライブラリがSMBのネットワークドライブ上だと、chokidarのイベントが届かないことがあります
- ヘッダー **[↻ 更新確認]** で手動取り込みできます([04章 4.10](04_便利機能.md#410-外部変更の取り込み更新確認))
- 5分間隔の定期ポーリングでも取り込まれます

## 7.10 それでも解決しない場合

- ブラウザのDevToolsコンソールにエラーが出ていないか確認
- サーバーログ(`LOG_FILE`)を確認(pino構造化ログ、`level: error` を絞ると絞りやすい)
- リポジトリのIssueとして報告: [GitHub Issues](https://github.com/kuninet/tsumiwiki/issues)
