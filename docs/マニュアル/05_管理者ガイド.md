# 05. 管理者ガイド

管理者(admin)ロールを持つユーザーだけができる操作をまとめます。ユーザーメニュー(右上のアバター)にadmin専用の項目が表示されます。

## 5.1 ユーザー管理

**右上のユーザーメニュー → [ユーザー管理]** で `/admin/users` を開きます。

### 5.1.1 ユーザー一覧

| 列 | 内容 |
|---|---|
| ID | 内部ID |
| ユーザーID | ログインに使う名前 |
| 表示名 | Gitコミットのauthor等に使われる名前 |
| ロール | 管理者 / 一般 |
| 状態 | 有効 / 無効 |
| 操作 | 表示名変更・昇格/降格・有効化/無効化・パスワードリセット |

### 5.1.2 ユーザーを追加する

1. 右上 **[+ ユーザー追加]** をクリック
2. 追加ダイアログ:
   - **ユーザーID** — ログイン名(重複不可)
   - **表示名** — 画面表示・Gitコミットに使われる名前
   - **パスワード** — 初期パスワード
   - **ロール** — 一般 / 管理者
3. **[追加]** で作成

### 5.1.3 表示名の変更

対象行 → **[表示名変更]** ボタン → 入力ダイアログ

### 5.1.4 パスワードリセット

対象行 → **[パスワードリセット]** → 新しいパスワードを入力

- ユーザーの現在のパスワードは知らなくてもリセットできます
- リセット直後、**対象ユーザーの全セッションが失効** します(強制ログアウト)
- **自分自身のパスワードはここでは変更できません**(理由: 変更した瞬間に自分のセッションも失効するため運用事故を防ぐ)
- 自分のパスワードは **個人設定** `/settings` から変更してください

### 5.1.5 ロールの変更(昇格/降格)

- 対象行 → **[昇格]** または **[降格]** ボタン
- 降格(admin → user)時は「管理者権限の解除」確認ダイアログを挟みます
- **最後のadminは降格できません**(誰も管理者がいなくなる状態を防ぎます)
- **自分自身の降格はできません**(自己ロックアウト防止)
- 降格時、対象ユーザーの全セッションが失効します

### 5.1.6 有効化・無効化

- 対象行 → **[無効化]** または **[有効化]** ボタン
- 無効化時は確認ダイアログを挟みます
- 無効化されたユーザーはログインできません(既存セッションも即失効)
- **自分自身の無効化はできません**(自己ロックアウト防止)

## 5.2 ライブラリ設定

**右上のユーザーメニュー → [ライブラリ設定]** で `/admin/library` を開きます。

このページの内容は `.tsumiwiki/settings.yaml` としてライブラリ内に保存され、Git履歴・バックアップpushにも自動で乗ります。

### 5.2.1 テンプレート設定

- **テンプレートフォルダ**(既定 `_templates`)
  - このフォルダ配下の `.md` がテンプレート機能([04章 4.1](04_便利機能.md#41-テンプレートから新規文書を作成する))で使えるようになります
  - 通常運用ではこの既定値でOKです

### 5.2.2 デイリーノート設定

- **作成先フォルダ**(例: `日記`, `daily`)
- **適用するテンプレート**(例: `_templates/日誌.md`。空欄で空白ノート)
- **ファイル名パターン**(例: `YYYY-MM-DD` → `2026-07-08.md`)
  - **日付書式のみ** 使えます。`{{...}}` 変数構文は不可(バリデーションエラー)
  - トークン: `YYYY / YY / MM / DD / HH / mm / ss / aaaa(月曜日) / aaa(月) / dddd(Monday) / ddd(Mon)`
  - 曜日付きの例: `YYYY-MM-DD(aaa)` → `2026-07-08(水).md`

### 5.2.3 保存

右下 **[保存]** ボタンで確定します。保存時は `config: update library settings` プレフィックスのGitコミットが1本作られます。

## 5.3 ゴミ箱の完全削除

**サイドバー → [🗑 ごみ箱]** で `/trash` を開きます。管理者には各行に **[完全削除]** ボタンが追加で表示されます。

- 「元に戻せません。よろしいですか?」の確認ダイアログを経て確定
- ファイル自体が削除され、Gitに `purge:` コミットが記録されます(履歴には残ります)

一般ユーザーには **[復元]** のみ表示されます([02章 2.10](02_基本操作.md#210-ゴミ箱復元))。

## 5.4 編集ロックの強制解除

現在このUIは実装されていません(将来対応)。緊急時はAPI経由で強制解除できます:

```
curl -X DELETE 'http://your-server:3000/api/locks/force?path=...' \
  -H 'X-Requested-With: TsumiWiki' \
  --cookie 'session=...'
```

将来的にはUI(ロック中の文書を右クリック等)から実行できるようにする予定です。

## 5.5 外部変更の取り込み(更新確認)

TsumiWikiは外部ツール(Obsidian・生成AIエージェント・直接ファイル操作)による変更を、以下の複数経路で自動的に取り込みます。

- **ファイル監視**(chokidar): 数秒〜数十秒
- **定期ポーリング**: 既定5分間隔
- **サーバー起動時の全走査**: 起動時
- **文書取得時のmtime不一致検知**: アクセスされた瞬間
- **手動**: ヘッダー **[↻ 更新確認]** ボタン

取り込まれた変更は `sync: external changes` プレフィックスのシステムコミットとして履歴に記録されます。

### ネットワークドライブでの注意

ライブラリがSMB等のネットワーク上にある場合、ファイル監視イベントが届かないことがあります。その場合も他の経路で数分以内に取り込まれます。

### 外部ツール利用者への推奨

- `.git/`・`.obsidian/`・`.trash/`・`.tsumiwiki/` を触らせない
- Wikiで誰かが編集中の文書は避けるか、AI用の作業フォルダを分ける
- 詳細は [連携/AIエージェント_直接編集ガイド](../連携/AIエージェント_直接編集ガイド.md) を参照

## 5.6 バックアップの確認

- 定期push間隔: `BACKUP_PUSH_INTERVAL_MINUTES`(既定10分)
- 状態確認: `GET /api/library/status` および `GET /api/health`(未来の警告UI用)
- push方式: `git push backup main --force-with-lease`
- push先: `BACKUP_REMOTE` 環境変数で指定(既定 bareリポジトリ)

バックアップ設定は環境変数で指定するため、変更時は **サーバーの再起動が必要** です([06章 セットアップ](06_セットアップ.md))。

## 5.7 復旧手順

### 5.7.1 ライブラリが失われた場合

1. bareリポジトリ(バックアップ先)から `git clone` でライブラリを復元:
   ```powershell
   git clone \\fileserver\share\tsumiwiki.git C:\tsumiwiki-library
   ```
2. TsumiWikiサーバーを再起動 — 自動でインデックスが再構築されます
3. ライブラリ設定(`.tsumiwiki/settings.yaml`)もクローンに含まれるため、追加の設定作業は不要です

### 5.7.2 SQLite(`app.db`)が失われた場合

文書自体は無傷です。以下で復旧します:

1. サーバー起動 — スキーマは自動作成されます
2. `tsumiwiki create-admin` で最初の管理者を再作成
3. `/admin/users` から他ユーザーを再登録
4. `tsumiwiki reindex` でフルリインデックス

ロック・下書きは失われますが、コミット済みの文書は残ります。

## 5.8 運用上の推奨

- 監査ログ(pino構造化ログ): 定期的に確認
- パスワードは初期発行後、ユーザー自身に変更してもらう
- 管理者は最低2名で運用(1名だと降格・無効化が全くできなくなる)
- バックアップpushの成功を月に一度は確認
