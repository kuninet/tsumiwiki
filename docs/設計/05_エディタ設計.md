# 05. エディタ設計

本システムの技術的中核。要件: FR-EDIT一式、FR-OBS(Obsidian互換)、FR-IMG(画像D&D)。

## 5.1 基本方針

1. **エディタはTiptap v2(ProseMirror)**。閲覧モードも同じスキーマで `editable: false` 描画し、閲覧/編集の見た目を一致させる。
2. **フロントマターはエディタに入れない**。`GET /api/docs` がフロントマターと本文を分離して返し、エディタは本文Markdownのみ扱う。タグ編集は専用UI(タグ入力欄)で行い、保存時にサーバーがフロントマターへ再結合する。往復変換の対象範囲を本文に限定し、破壊リスクを下げる(FR-EDIT-06)。
3. **往復変換の保証はテストで担保**。実ヴォルト由来のサンプル文書で「parse→serialize が冪等(2回目以降変化しない)」を検証する回帰テストを整備する。

## 5.2 Markdown ⇔ ProseMirror 変換

```
読み込み: Markdown(本文) ─ markdown-it + 独自拡張 ─▶ ProseMirrorドキュメント
保存:     ProseMirrorドキュメント ─ 独自シリアライザ ─▶ Markdown(本文)
```

- ベースに `tiptap-markdown`(markdown-it利用)を採用し、独自ノードの変換規則を追加する。
- 対応記法(FR-EDIT-02)と変換規則:

| 記法 | ProseMirrorノード | シリアライズ規則 |
|---|---|---|
| 見出し `#`〜`######` | heading | `#` 形式固定 |
| 太字/斜体/打消し | bold/italic/strike | `**` `*` `~~` |
| 箇条書き/番号付き | bulletList/orderedList | `-` / `1.` |
| チェックリスト | taskList/taskItem | `- [ ]` / `- [x]` |
| 表(GFM) | table | GFMパイプ表 |
| コードブロック | codeBlock(language属性) | フェンス ``` 言語名 |
| 引用 | blockquote | `>` |
| 水平線 | horizontalRule | `---` |
| リンク | link mark | `[text](url)` |
| 画像(標準) | image | `![alt](相対パス)` |

- **未知の記法の保全**: パースできないブロック(HTMLブロック等)は `rawBlock` ノード(等幅・グレー背景の編集可能テキスト)として保持し、シリアライズ時に原文をそのまま出力する。「読んだものは壊さない」を最優先とする。

## 5.3 Obsidian互換の独自拡張

### wikilink(FR-OBS-02)

- インラインノード `wikilink { target, alias }`。正規表現 `[[target]]` / `[[target|alias]]` をmarkdown-itインラインルールで認識。
- 表示: リンク風スタイル。クリックで `target` を解決し(`/api/tree` のタイトル索引から最短パス規則で一致)、該当文書へ遷移。未解決はグレー表示(遷移不可。リンク切れ検出 FR-LINK-03 の土台)。
- 入力: `[[` とタイプするとTiptap Suggestion による文書名補完ポップアップを表示(FR-LINK-01)。
- シリアライズ: 原文どおり `[[target]]` / `[[target|alias]]`。

### 埋め込み `![[...]]`(FR-OBS-03)

- ブロック/インラインノード `embed { target }`。画像拡張子(`png/jpg/jpeg/gif/svg/webp`)ならファイル名をライブラリ索引から解決し `/api/files/<path>` を `<img>` 表示。
- 画像以外(文書埋め込み等)は初期リリースではリンクとして表示(展開しない)。シリアライズは原文保持。

### インラインタグ(FR-OBS-06)

- 編集中は装飾(Decoration)で `#タグ` をハイライト表示するのみ。ノード化しない(プレーンテキストのまま保存されるため往復安全)。
- タグの抽出・集計はサーバーのインデックスサービスが担当(02章2.3)。

### Mermaid(FR-EDIT-10)

- `codeBlock(language=mermaid)` にNodeViewを適用:
  - 閲覧モード: mermaid.jsでSVGレンダリング(`securityLevel: 'strict'`)。構文エラー時はエラーメッセージ+ソース表示。
  - 編集モード: フォーカス中はソース編集、フォーカスが外れるとプレビュー表示に戻る。
- シリアライズ: 通常のコードブロックとして原文出力。

### Dataviewブロック(FR-OBS-08)

- `codeBlock(language=dataview|dataviewjs)` は**実行せず**通常のコードブロック表示。専用NodeView不要。往復で原文保持されることをテスト対象に含める。

### 外部ファイルリンク(FR-LINK-02)

- `file://` スキームまたはUNCパス(`\\server\...`)のリンクは、閲覧時にリンク横へ「パスをコピー」ボタンを表示するNodeViewを適用。クリックによる直接遷移も妨げない(許可された環境向け)。

## 5.4 ツールバー(FR-EDIT-03)

```
[H1 H2 H3] [B I S] [リスト 番号 チェック] [表] [コード 引用 区切り線] [リンク 画像] [Mermaid]
```

- 各ボタンはTiptapコマンドを呼ぶ。現在のカーソル位置の状態をボタンのアクティブ表示に反映。
- 「画像」はファイル選択ダイアログ→添付アップロード(D&Dと同経路)。
- 「Mermaid」は空のmermaidコードブロックを挿入するテンプレートボタン。

## 5.5 画像の挿入経路(FR-IMG-01/02)

```
D&D / ペースト / ツールバー
  → POST /api/attachments (multipart, docPath)
  → サーバー: 文書と同フォルダに image-YYYYMMDDHHmmss.png で保存+コミット
  → 応答のファイル名で ![[ファイル名]] ノードを挿入
```

- アップロード中はプレースホルダ表示。失敗時はノードを除去しトースト表示。
- 対応形式: png / jpg / jpeg / gif / svg / webp。サイズ上限20MB(設定可)。

## 5.6 日本語入力(IME)対応(FR-EDIT-05)

- ProseMirrorのcomposition対応に準拠。追加規則:
  - 入力規則(`#` →見出し等のInput Rules)・サジェスト発火は `compositionend` 後にのみ評価。
  - ショートカット処理は `event.isComposing` 中は無視。
- 受け入れ確認: macOS(ライブ変換オン)+ Windows(MS-IME)で、変換中文字列の分断・確定取り消し・wikilink補完の挙動を手動テストする(検証項目リストを実装時に作成)。

## 5.7 往復変換テスト(FR-EDIT-06の担保)

- `packages/client/src/editor/__tests__/roundtrip/` にサンプルMarkdown(実ヴォルトから抜粋・匿名化)を配置。
- テスト: `serialize(parse(md))` が `md` と一致(初回は正規化差分を許容し、2回目 `serialize(parse(x)) === x` の冪等性を必須とする)。
- CI(GitHub Actions)で常時実行。対応記法を追加するたびにサンプルを追加する。
