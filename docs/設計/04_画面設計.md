# 04. 画面設計

## 4.1 ルーティングと画面遷移

SPA(React Router)。要件定義04章の画面一覧に対応する。

| ルート | 画面 | 備考 |
|---|---|---|
| `/login` | ログイン(SC-01) | 未認証時は全ルートからここへリダイレクト |
| `/` | メイン(SC-02) | 文書未選択状態(最近更新一覧を表示) |
| `/doc/*` | メイン(SC-02) | `*` は文書パス。閲覧/編集モードは画面内state |
| `/trash` | ごみ箱(SC-07) | |
| `/admin/users` | ユーザー管理(SC-05) | adminのみ。他ロールは403画面 |
| `/admin/library` | ライブラリ設定 | adminのみ。テンプレフォルダ・デイリーノート3項目を編集 |
| `/settings` | 個人設定(SC-06) | パスワード変更 |
| `/demo` | エディタデモ | 左WYSIWYG/右Markdownシリアライズの確認用(開発補助) |

- 履歴(SC-03)はメイン画面のサイドパネル、検索結果(SC-04)はヘッダー検索ボックスからのドロップダウン+一覧パネルとして実装(独立ルートにしない)。
- 文書パスをURLに持たせることで、文書へのブラウザブックマーク・リンク共有を可能にする。

## 4.2 コンポーネント構成

```
<App>
 ├─ <LoginPage>
 └─ <AppShell>                        … 認証済みレイアウト
     ├─ <Header>                      … ロゴ / <SearchBox> / [↻ 更新確認] / <ThemeToggle> / <UserMenu>
     ├─ <Sidebar>                     … 幅ドラッグ(200〜480px)・折りたたみ可
     │   ├─ <SidebarTabs>             … [フォルダ | タグ]
     │   ├─ <FolderTree>              … ツリー+コンテキストメニュー+複数選択
     │   ├─ <TagPane>                 … タグ一覧(件数)・AND絞り込み
     │   └─ <SidebarFooter>           … [📓今日の日誌][📄テンプレから新規][+新規文書][🗑ごみ箱]
     ├─ <MainPane>
     │   ├─ <DocHeader>               … タイトル・更新情報・[履歴][編集/破棄/保存]・ロック表示・<TagChipEditor>
     │   ├─ <DocView>                 … 閲覧/編集モード(Tiptap を editable 切替。<EditorToolbar>)
     │   ├─ <HistoryPanel>            … 履歴一覧・<DiffView>・復元
     │   └─ <RecentDocs> / <SearchResults> / <TagResults>
     ├─ <TemplatePickerDialog>        … create / apply の2モード(2段UI)
     └─ <StatusBar>                   … 閲覧/編集/保存状態・ロック状態
```

- `<DocView>` は Tiptap を `editable` 切替で使い、閲覧/編集で描画を完全に一致させる(モード切替で表示が変わらない)。**文書を開いた瞬間に編集モードに入る**(他者ロック中は閲覧モードにフォールバック)(#51)。
- `<TagChipEditor>`: 閲覧時はクリックでタグ絞り込み、編集時は `#タグ` のクリックでインラインrename、× で削除、末尾の「+ タグを追加」で追加。frontmatter と本文中 `#タグ` の両方を編集対象とする。「+ タグを追加」の入力欄直下には `useTags()` から取得した既存タグの候補ポップアップを表示し(付与済みタグは除外、入力値で部分一致フィルタ、count降順の上位20件)、クリックまたは↑↓+Enterで選択確定できる(#118)。
- `<TemplatePickerDialog>`:
  - `create` モード(サイドバーフッター「テンプレから新規」から起動): Step1 テンプレ選択、Step2 タイトル+作成先フォルダ → `POST /api/templates/apply` → 新規文書へ遷移。
  - `apply` モード(編集中ツールバー「テンプレ適用」から起動): Step2 で [挿入] / [追記] を選択 → `POST /api/templates/expand` → エディタに1トランザクションで流し込む(1回の Undo で完全復帰)。`{{cursor}}` があれば挿入後にその位置へカーソル移動。
- コンテキストメニュー操作: 新規文書・新規フォルダ・リネーム・複数選択→新規フォルダにまとめる・削除(確認ダイアログ→ごみ箱へ)。
- フォルダツリーの複数選択: Ctrl/Cmd+クリックで個別トグル、Shift+クリックで範囲選択、D&D で一括移動、右クリック→「選択したものを新規フォルダに移動」で一発グルーピング。

## 4.3 状態管理

| 状態 | 管理 | 内容 |
|---|---|---|
| サーバー状態 | TanStack Query | tree / doc / tags / search / history / trash / users。保存・削除等のmutation後に該当クエリをinvalidate |
| UI状態 | Zustand | サイドバー幅・選択タブ・編集モードフラグ・未保存フラグ(dirty)・選択中タグ |
| エディタ状態 | Tiptap内部 | ドキュメント本体。dirty判定はTiptapの `onUpdate` で立てる |

- ロック表示の鮮度: 文書表示中は `GET /api/docs` を60秒間隔でrefetchし、ロック状態を追随(リアルタイム性は要求しない)。

## 4.4 編集モードのライフサイクル

```
文書オープン ──▶ ロック取得POST ──成功──▶ 編集モードで開く(#51)
                        │失敗(DOC_LOCKED)
                        ▼
                「◯◯さんが編集中」トースト・閲覧モードにフォールバック

編集中: 60秒ごとロックrefresh(ハートビート) / 30秒ごと下書きPUT(dirty時のみ)
   │
   ├─ [✓ 保存](Ctrl/Cmd+S) ─▶ PUT /api/docs ─▶ 編集モード継続(#51)
   │                        競合(baseUpdatedAt不一致): 「保存の競合」ダイアログ
   │                          → [自分の内容で上書き保存] / [破棄して最新を読み込む]
   ├─ [破棄] ─▶ 確認 ─▶ 下書き破棄・ロック解放・閲覧モードへ
   └─ ページ離脱 ─▶ beforeunloadで警告(FR-EDIT-07)+ pagehideでkeepalive fetchによる
                    「最終下書き保存 → ロック解放」を非同期送信
```

- クラッシュ復帰: 編集開始時に自分の下書きが残っていれば「未保存の下書きがあります。復元しますか?」ダイアログで [復元] / [破棄] を選択。
- ロック失効(サーバー側で自動解放された場合): 次のハートビートで検知し、「ロック期限切れ」トーストで閲覧モードに戻す。
- SPA内遷移時の未保存確認: `confirmNavigationIfDirty()` により、他ページへの遷移時もブラウザ標準の離脱確認を表示する。

## 4.5 キーボードショートカット(FR-EDIT-04)

| キー | 動作 | スコープ |
|---|---|---|
| Ctrl/Cmd+S | 保存 | 編集中(ブラウザ既定を抑止) |
| Ctrl/Cmd+B / I / U | 太字 / 斜体 / 下線 | 編集中(Tiptap StarterKit 由来) |
| Ctrl/Cmd+K | リンク挿入 | 編集中 |
| Ctrl/Cmd+K | 検索ボックスへフォーカス | 閲覧中(グローバル) |
| Ctrl/Cmd+Z / Shift+Z | 取り消し / やり直し | 編集中 |
| ↑ / ↓ | フォーカス/候補移動 | フォルダツリー・検索候補・wikilink候補・テンプレピッカー |
| → / ← | フォルダ展開 / 折り畳み(またはアウトデント) | フォルダツリー |
| Enter | 選択決定・文書オープン | 検索候補・ツリー・候補ポップアップ |
| Escape | ダイアログ/検索候補/履歴パネル/wikilink候補を閉じる | 全般 |
| F2 | リネーム | フォルダツリー行 |
| Delete | ごみ箱へ移動 | フォルダツリー行 |
| Backspace | 空リスト項目末で解除→段落 | 編集中(Obsidian互換) |
| `[[` | wikilink候補ポップアップを開く | 編集中(IME中は無視) |
| Ctrl/Cmd+P | 文書名クイックオープン | **未実装(将来検討)** |

- 実装: 編集系はTiptapのkeymap、グローバル系は`useHotkeys`相当のフックで管理し、IME変換中(composition中)は発火させない。

## 4.6 表示・スタイル方針

- 最小幅1280pxのPC向け2カラム(サイドバー260px既定+本文)。サイドバーは200〜480pxでリサイズ可、折り畳み可。
- **ライト/ダーク両対応**。zustand persistで選択を保持し、`<html data-theme="light|dark">` 属性で切り替える。初期リリースはライトのみだったが、Phase 1 で ThemeToggle を追加した。
- CSSはTailwind CSSを採用。デザインシステムは `docs/デザイン/handoff/` のトークンに従う。
- 日本語フォントはシステムフォントスタック(游ゴシック/ヒラギノ/Noto Sans JP)。
- トースト通知(保存完了・エラー)と確認ダイアログを共通コンポーネント化。エラー種のトーストのみユーザーが × で明示的に閉じる仕様(見逃し防止)。
