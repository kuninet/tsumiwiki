# 04. 画面設計

## 4.1 ルーティングと画面遷移

SPA(React Router)。要件定義04章の画面一覧に対応する。

| ルート | 画面 | 備考 |
|---|---|---|
| `/login` | ログイン(SC-01) | 未認証時は全ルートからここへリダイレクト |
| `/` | メイン(SC-02) | 文書未選択状態(最近更新一覧を表示) |
| `/doc/*` | メイン(SC-02) | `*` は文書パス。閲覧/編集モードは画面内state |
| `/trash` | ごみ箱(SC-07) | |
| `/admin/users` | ユーザー管理(SC-05) | adminのみ。他ロールは403画面 |
| `/settings` | 個人設定(SC-06) | パスワード変更 |

- 履歴(SC-03)はメイン画面のサイドパネル、検索結果(SC-04)はヘッダー検索ボックスからのドロップダウン+一覧パネルとして実装(独立ルートにしない)。
- 文書パスをURLに持たせることで、文書へのブラウザブックマーク・リンク共有を可能にする。

## 4.2 コンポーネント構成

```
<App>
 ├─ <LoginPage>
 └─ <AppShell>                        … 認証済みレイアウト
     ├─ <Header>                      … ロゴ / <SearchBox> / ユーザーメニュー
     ├─ <Sidebar>                     … 幅ドラッグ・折りたたみ可
     │   ├─ <SidebarTabs>             … [フォルダ | タグ]
     │   ├─ <FolderTree>              … ツリー+コンテキストメニュー
     │   └─ <TagPane>                 … タグ一覧(件数)・AND絞り込み
     ├─ <MainPane>
     │   ├─ <DocHeader>               … タイトル・更新情報・[履歴][編集/保存]・ロック表示
     │   ├─ <DocViewer>               … 閲覧モード(読み取り専用エディタ)
     │   ├─ <DocEditor>               … 編集モード(<EditorToolbar> + Tiptap)
     │   ├─ <HistoryPanel>            … 履歴一覧・<DiffView>・復元
     │   └─ <RecentDocs> / <SearchResults> / <TagResults>
     └─ <StatusBar>                   … 保存状態・ロック状態
```

- `<DocViewer>` は Tiptap を `editable: false` で使い、閲覧/編集で描画を完全に一致させる(モード切替で表示が変わらない)。
- コンテキストメニュー操作: 新規文書・新規フォルダ・リネーム・移動・削除(確認ダイアログ→ごみ箱へ)。

## 4.3 状態管理

| 状態 | 管理 | 内容 |
|---|---|---|
| サーバー状態 | TanStack Query | tree / doc / tags / search / history / trash / users。保存・削除等のmutation後に該当クエリをinvalidate |
| UI状態 | Zustand | サイドバー幅・選択タブ・編集モードフラグ・未保存フラグ(dirty)・選択中タグ |
| エディタ状態 | Tiptap内部 | ドキュメント本体。dirty判定はTiptapの `onUpdate` で立てる |

- ロック表示の鮮度: 文書表示中は `GET /api/docs` を60秒間隔でrefetchし、ロック状態を追随(リアルタイム性は要求しない)。

## 4.4 編集モードのライフサイクル

```
閲覧 ── [編集] ──▶ ロック取得POST ──成功──▶ 編集モード
                        │失敗(DOC_LOCKED)
                        ▼
                「◯◯さんが編集中」トースト表示・閲覧のまま

編集中: 60秒ごとロックrefresh / 30秒ごと下書きPUT(dirty時のみ)
   │
   ├─ [保存](Ctrl+S) ─▶ PUT /api/docs ─▶ 閲覧モードへ・ロック解放
   ├─ [キャンセル] ─▶ 確認 ─▶ 下書き破棄・ロック解放・閲覧モードへ
   └─ ページ離脱 ─▶ beforeunloadで警告(FR-EDIT-07)+ sendBeaconでロック解放を試行
```

- クラッシュ復帰: 編集開始時に自分の下書きが残っていれば「未保存の下書きがあります。復元しますか?」を表示。

## 4.5 キーボードショートカット(FR-EDIT-04)

| キー | 動作 | スコープ |
|---|---|---|
| Ctrl/Cmd+S | 保存 | 編集中(ブラウザ既定を抑止) |
| Ctrl/Cmd+B / I | 太字 / 斜体 | 編集中 |
| Ctrl/Cmd+K | リンク挿入 | 編集中 |
| Ctrl/Cmd+Z / Shift+Z | 取り消し / やり直し | 編集中 |
| Ctrl/Cmd+P | 文書名クイックオープン(COULD) | 全体 |

- 実装: 編集系はTiptapのkeymap、グローバル系は`useHotkeys`相当のフックで管理し、IME変換中(composition中)は発火させない。

## 4.6 表示・スタイル方針

- 最小幅1280pxのPC向け2カラム(サイドバー260px既定+本文)。
- ライトテーマのみ(初期リリース)。CSSはTailwind CSSを採用。
- 日本語フォントはシステムフォントスタック(游ゴシック/ヒラギノ/Noto Sans JP)。
- トースト通知(保存完了・エラー)と確認ダイアログを共通コンポーネント化。
